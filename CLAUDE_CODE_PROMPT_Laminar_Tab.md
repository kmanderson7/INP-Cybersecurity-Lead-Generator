## 1. Business Context (READ FIRST — IT DRIVES EVERY DECISION)

Laminar Digital is a programmable post-trade settlement rail for physical crude oil. It replaces the **Letter of Credit (LC) process** — a 20-to-30-day, $25K-to-$100K-per-cargo paper procedure — with **atomic, condition-based settlement** that releases escrow the moment delivery is verified.

### What we sell

**Working capital.** Not technology. Not a network. Not a platform.

A trading company running 50 cargoes per year at $5M average has approximately **$75M in working capital locked** in settlement cycles at any moment. A bank whose LC-backed credit facility turns over in 2 days instead of 20 days can support **10x the transaction volume** on the same balance sheet. That math is the entire pitch. Every component you build must surface it.

### Who buys

- **CFOs and Treasurers** at energy companies (Vitol, Mercuria, Trafigura, Gunvor + regional houses)
- **VP / MD / Head of Trade Finance** at banks (BNY Mellon, ING, Société Générale, Crédit Agricole, MUFG)
- **Operations Directors** at commodity trading houses
- **Chief Risk Officers** at all of the above
- **Pipeline / terminal operators** (Enterprise Products, Energy Transfer, Magellan, Buckeye)
- **Inspection company leadership** (SGS, Bureau Veritas, Intertek)

These buyers do not read crypto whitepapers. They read 10-Ks and quarterly earnings calls. They care about basis points on credit facilities and days-of-sales-outstanding on settlement cycles.

### The four pillars (UI motif throughout)

Settlement releases when all four conditions verify:

1. **Pipeline Nomination confirmed** (terminal/pipeline operator)
2. **Storage Verified** (terminal storage tank certificate)
3. **Inspection Certificate submitted** (independent inspector — SGS / Bureau Veritas / Intertek)
4. **Bill of Lading received** (carrier / vessel master)

Every company has a "readiness" status across these four pillars (compatible / likely / unknown / incompatible). The UI surfaces this on the company Overview tab.

### Linguistic guardrails (ABSOLUTE — VIOLATION INVALIDATES THE OUTPUT)

In **all user-facing copy** generated by AI or written into UI strings:

| ❌ Never use | ✅ Use instead |
|--------------|----------------|
| crypto, cryptocurrency | (omit — this is finance, not crypto) |
| blockchain | "programmable settlement" or "settlement rail" |
| XRP, XRPL, ledger | (omit — call it "the network" if needed) |
| NFT | "digital title document" |
| token | "settlement instrument" or "settlement unit" |
| DeFi, Web3, digital asset | (omit) |

This applies to outreach copy, AI insight cards, decision cards, working-capital narratives, system prompts, error messages, and tooltips. The Outreach Generator must include a client-side guardrail check that flags violations.

#### 1.1 `computeContactHeat(contact, parentCompany)`
```js
export function computeContactHeat(contact, company) {
  const baseRel = Number(contact?.relevanceScore || 0);
  const baseScore = Number(company?.leadScore || 0);

  let recencyBoost = 0;
  const signals = Array.isArray(company?.signals) ? company.signals : [];
  const now = Date.now();
  for (const sig of signals) {
    const occurredAt = sig?.occurredAt ? new Date(sig.occurredAt).getTime() : null;
    if (!occurredAt) continue;
    const days = (now - occurredAt) / (24 * 60 * 60 * 1000);
    if (days <= 7) recencyBoost = Math.max(recencyBoost, 20);
    else if (days <= 30) recencyBoost = Math.max(recencyBoost, 10);
  }

  const heat = Math.round(0.5 * baseRel + 0.3 * baseScore + recencyBoost);
  return Math.max(0, Math.min(100, heat));
}
```

```js
export function computeWorkingCapital(annualCargoes, avgCargoValue, settlementDays = 20) {
  const locked = Math.round((annualCargoes * avgCargoValue * settlementDays) / 365);
  const lcCostLow = Math.round(annualCargoes * avgCargoValue * 0.005);
  const lcCostHigh = Math.round(annualCargoes * avgCargoValue * 0.02);
  return { locked, lcCostLow, lcCostHigh };
}
```

```js
import { errorResponse, successResponse } from '../lib/http.js';
import { checkRateLimit } from '../lib/rateLimit.js';
import { requireLiveDataEnabled } from '../lib/source.js';
import OpenAI from 'openai';

const PROMPTS = {
  executiveInsight: { system: '…', userTemplate: '…' },
  outreach: { system: '…', userTemplate: '…' },
  decisionCards: { system: '…', userTemplate: '…' },
  workingCapitalNarrative: { system: '…', userTemplate: '…' },
  explainScore: { system: '…', userTemplate: '…' }
};

export async function handler(event) {
  if (event.httpMethod === 'OPTIONS') return { statusCode: 204, headers: {/*CORS*/} };
  if (event.httpMethod !== 'POST') return errorResponse('Method Not Allowed', 405);

  const clientIP = event.headers['x-forwarded-for'] || 'anonymous';
  const rate = checkRateLimit(`laminar_ai_${clientIP}`, 30, 60 * 60 * 1000);
  if (!rate.allowed) return errorResponse('Rate limit exceeded', 429);

  try {
    const { feature, payload } = JSON.parse(event.body || '{}');
    const prompt = PROMPTS[feature];
    if (!prompt) return errorResponse(`Unknown feature: ${feature}`, 400);

    if (!process.env.OPENAI_API_KEY) {
      if (requireLiveDataEnabled()) {
        return errorResponse('OPENAI_API_KEY not configured', 503, {
          source: 'provider_fallback',
          provider: 'openai',
          reason: 'REQUIRE_LIVE_DATA blocked Laminar AI fallback.'
        });
      }
      // Non-live mode: return template-based fallback per feature
      return successResponse({ result: buildFallback(feature, payload) }, {
        source: 'provider_fallback',
        provider: 'laminar_ai_fallback',
        confidence: 0.3
      });
    }

    const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });
    const userMessage = renderTemplate(prompt.userTemplate, payload);
    const completion = await openai.chat.completions.create({
      model: 'gpt-4o-mini',
      max_tokens: 1000,
      messages: [
        { role: 'system', content: prompt.system },
        { role: 'user', content: userMessage }
      ]
    });

    return successResponse({ result: completion.choices[0].message.content }, {
      source: 'provider_live',
      provider: 'openai',
      confidence: 0.85
    });
  } catch (error) {
    console.error('[laminar-ai] error:', error);
    return errorResponse(error.message || 'Laminar AI dispatch failed', 500);
  }
}
```
